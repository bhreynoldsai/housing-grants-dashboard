# Project notes for Claude Code

**What this is:** A Vite + React + Tailwind dashboard for searching housing grants
and programs (federal, state, local) by city/county, distinguishing rural vs. urban
eligibility. Built for the Rural Prosperity HUB Initiative but generalizable.

**Key files:**
- `src/App.jsx` — the whole UI (search bar, filters, results list, detail modal,
  shortlist sidebar, live-opportunities panel). Currently a single component file
  by design (small project); split into `src/components/*.jsx` if it grows.
- `src/data/programs.js` — hand-curated dataset of ~29 programs. This is the
  source of truth for the main results list.
- `src/data/liveOpportunities.json` — machine-generated by
  `scripts/fetch-grants.mjs`; do not hand-edit, it gets overwritten.
- `scripts/fetch-grants.mjs` — Node script hitting Grants.gov's public,
  unauthenticated `search2` API. Run via `npm run fetch-grants`.
- `scripts/fetch-hud-entitlements.mjs` — Node script pulling HUD's real CDBG/HOME
  entitlement jurisdiction list from HUD's ArcGIS Open Data site (resolves item
  IDs to service URLs via Esri's Sharing API, then paginates the attribute
  query). Run via `npm run fetch-hud`. Output feeds `classifyLocation()` in
  `App.jsx`, which prefers this live dataset over the static
  `GA_ENTITLEMENT`/`MO_ENTITLEMENT` arrays whenever it's populated.

**Likely next tasks a user might ask for:**
1. *"Add support for [other state]"* — add a `{STATE}_ENTITLEMENT` array in
   `programs.js` following the `GA_ENTITLEMENT`/`MO_ENTITLEMENT` pattern, add it
   to `STATIC_ENTITLEMENT` in `App.jsx`, add the option to the state `<select>`,
   and (optionally) add that state's housing-finance-agency programs with a
   matching `stateTag`. `classifyLocation()` already branches generically on
   `state` rather than hardcoding Georgia. Note `fetch-hud-entitlements.mjs`
   already covers every state nationwide once run — a new state may need zero
   code changes to its *entitlement classification* if you're fine relying on
   the live dataset alone.
2. *"Add a map view"* — city/county entries don't currently carry lat/lng; you'd
   need a geocoding step (either a static county-centroid lookup table checked
   into `src/data/`, or a client-side geocoding API call).
3. *"Deploy this"* — it's a fully static Vite build (`npm run build` → `dist/`),
   deployable to Vercel/Netlify/GitHub Pages/S3 with no server component.
4. *"The live opportunities panel is empty / wrong fields"* — run
   `npm run fetch-grants`, then inspect `src/data/_debug-raw-sample.json` (the raw
   Grants.gov response) and adjust the field-name guesses in `mapHit()` inside
   `scripts/fetch-grants.mjs`.

**Conventions to preserve:**
- Palette: navy `#0B1F3A` / slate `#1E3A5F` / gold `#C9A227` / parchment
  `#F5F2EA` background — matches the navy/blue/gold palette used across this
  project's other deliverables (proposals, memos). Defined in `tailwind.config.js`.
- Every program object in `programs.js` should carry `sourceUrl` and
  `lastVerified` — this is a grant-strategy tool for a real submission, so
  provenance matters more than in a typical demo app.
